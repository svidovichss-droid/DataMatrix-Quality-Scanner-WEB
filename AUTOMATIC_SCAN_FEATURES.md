# Реализация автоматического сканирования Data Matrix с историей и звуковым сопровождением

## Обзор изменений

В систему сканирования Data Matrix добавлены следующие функции:

### 1. Запись истории сканирования
- **Местоположение**: `src/ui/main_window.py`
- **Реализация**: 
  - Массив `scanned_codes_history` хранит последние 1000 отсканированных кодов
  - Ключ дубликата: `{barcode_data}_{position}` для предотвращения повторного сканирования
  - Автоматическая очистка при очистке результатов через `_on_clear_results()`

### 2. Сканирование в постоянном режиме
- **Местоположение**: `src/ui/main_window.py`, `scanner/quality_analyzer.py`
- **Реализация**:
  - Кнопка "ЗАПУСТИТЬ ИНСПЕКЦИЮ" активирует непрерывный режим
  - Обработка каждого кадра через `process_frame()` в `ConveyorAnalyzer`
  - Последовательность операций:
    1. Поиск DataMatrix (детектирование/локализация)
    2. Захват области интереса (ROI)
    3. Сканирование (декодирование)
    4. Верификация качества по ГОСТ Р 57302-2016

### 3. Захват Data Matrix
- **Местоположение**: `scanner/datamatrix_decoder.py`, `scanner/quality_analyzer.py`
- **Реализация**:
  - Метод `detect_codes()` - поиск и локализация кодов в кадре
  - Метод `decode_region()` - декодирование захваченной области
  - Автоматическое выделение ROI с отступом 20 пикселей
  - Поддержка нескольких методов детектирования:
    - pylibdmtx (основной)
    - pyzbar (альтернативный)
    - OpenCV WeChat QR detector
    - Детекция через контуры

### 4. Анализ Data Matrix
- **Местоположение**: `scanner/quality_analyzer.py`
- **Реализация**:
  - Верификация по параметрам ГОСТ Р 57302-2016 (ISO/IEC 15415):
    - Decode (декодируемость)
    - Symbol Contrast (контраст символа)
    - Min Reflectance (минимальное отражение)
    - Min Edge Contrast (контраст края)
    - Modulation (модуляция)
    - Defects (дефекты)
    - Decodability (возможность декодирования)
  - Оценка по шкале A (4.0) - F (0.0)
  - Статус PASS/FAIL на основе минимального порога

### 5. Вывод результата со звуковым сопровождением
- **Местоположение**: `src/utils/sound_notifier.py`, `src/ui/main_window.py`
- **Реализация**:
  - Класс `SoundNotifier` поддерживает несколько платформ:
    - Windows: winsound.Beep()
    - Linux/Mac: системный beep ('\a')
    - PyQt6: QSoundEffect (при наличии Qt Multimedia)
  - Два типа звуковых сигналов:
    - `play_success()`: высокий тон (1200 Гц) при успешном сканировании
    - `play_failure()`: низкий тон (400 Гц) при ошибке
  - Звук воспроизводится в `_on_inspection_result()` после проверки на дубликат

### 6. Автоматический режим
- **Местоположение**: `src/ui/main_window.py`
- **Реализация**:
  - После запуска инспекции система работает автоматически
  - Каждый кадр обрабатывается последовательно
  - Результаты отображаются в реальном времени:
    - Таблица результатов с цветовой индикацией
    - Текущий результат с деталями оценки
    - Лог событий
    - Статистика с прогресс-барами
  - Защита от дубликатов предотвращает повторную обработку одного кода

## Использование

1. **Запуск приложения**:
   ```bash
   python src/main.py
   ```

2. **Настройка**:
   - Выберите камеру из списка
   - Настройте разрешение и FPS
   - Установите минимальный порог качества (по умолчанию 2.0 = C)

3. **Работа в автоматическом режиме**:
   - Нажмите "Подключить" для инициализации камеры
   - Нажмите "ЗАПУСТИТЬ ИНСПЕКЦИЮ" для начала непрерывного сканирования
   - Система автоматически:
     - Обнаруживает Data Matrix коды в кадре
     - Захватывает область с кодом
     - Декодирует данные
     - Анализирует качество по ГОСТ
     - Воспроизводит звуковой сигнал
     - Записывает результат в историю и таблицу
     - Продолжает сканирование следующего кода

4. **Просмотр результатов**:
   - Вкладка "Результаты": таблица всех сканирований
   - Вкладка "Статистика": распределение оценок
   - Вкладка "Управление": лог событий и текущий результат

5. **Экспорт данных**:
   - Кнопка "Экспорт CSV": сохранение результатов в CSV
   - Кнопка "Сгенерировать отчет": JSON отчет со статистикой

## Технические детали

### Защита от дубликатов
```python
code_key = f"{result.barcode_data}_{result.position}"
if code_key in self.scanned_codes_history:
    return  # Пропуск дубликата
self.scanned_codes_history.append(code_key)
if len(self.scanned_codes_history) > 1000:
    self.scanned_codes_history.pop(0)
```

### Звуковое сопровождение
```python
if self.sound_notifier:
    if result.passed:
        self.sound_notifier.play_success()  # Высокий тон
    else:
        self.sound_notifier.play_failure()  # Низкий тон
```

### Логирование в CSV
Автоматически создается файл `reports/inspection_log_<timestamp>.csv` с колонками:
- Timestamp, Barcode Data, Overall Grade, Decode, Symbol Contrast,
  Min Reflectance, Min Edge Contrast, Modulation, Defects, Decodability,
  Position X, Position Y, Passed, Image Path

## Требования

- Python 3.8+
- opencv-python-headless
- pylibdmtx
- pyzbar
- PyQt6
- libegl1, libgl1-mesa-glx (для Linux)

## Примечания

- Для работы в headless-среде используйте виртуальный дисплей (Xvfb)
- Звуковое сопровождение требует наличия аудиоустройства или терминала с поддержкой beep
- Для промышленного использования рекомендуется настроить внешний триггер конвейера
