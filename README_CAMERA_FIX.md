# Исправление проблемы с распознаванием веб-камеры

## Что было исправлено

### 1. scanner/camera_capture.py
- **Автоматическое определение платформы**: Код теперь определяет ОС (Windows/Linux/macOS) и выбирает правильный бэкенд OpenCV
- **Приоритизация бэкендов**:
  - Windows: CAP_DSHOW → CAP_MSMF → CAP_ANY → CAP_V4L2
  - Linux: CAP_V4L2 → CAP_ANY → CAP_FFMPEG
  - macOS: CAP_ANY → CAP_AVFOUNDATION → CAP_FFMPEG
- **Обработка ошибок**: Добавлен try-catch для каждого бэкенда
- **Улучшенное логирование**: Выводится информация о том, какой бэкенд использовался

### 2. src/ui/main_window.py
- **Автосканирование камер**: При запуске приложение автоматически сканирует все доступные камеры (индексы 0-9)
- **Отображение информации**: В выпадающем списке показывается разрешение каждой найденной камеры
- **Блокировка при отсутствии камер**: Если камеры не найдены, кнопка "Подключить" блокируется
- **Правильный выбор камеры**: Используется `currentData()` вместо `currentIndex()` для получения правильного ID

## Как использовать

### На Windows:
```bash
python src/main.py
```

### На Linux:
```bash
python src/main.py
```

### Диагностика проблем:

1. Проверьте список доступных камер в приложении
2. Если камеры не найдены:
   - Убедитесь, что камера подключена
   - Проверьте драйверы камеры
   - Убедитесь, что камера не используется другим приложением
   - На Linux проверьте права доступа: `ls -l /dev/video*`

### Тестовый скрипт:
Для проверки камеры вне приложения:
```bash
python /tmp/test_camera.py [camera_id]
```

## Возможные проблемы и решения

### Камера не определяется на Linux:
```bash
# Проверьте наличие устройства
ls -l /dev/video*

# Проверьте права доступа
groups $USER  # должна быть группа video

# Добавьте пользователя в группу video
sudo usermod -a -G video $USER
```

### Камера не определяется на Windows:
- Установите драйверы камеры
- Проверьте в Диспетчере устройств
- Попробуйте разные порты USB

### Камера занята другим приложением:
- Закройте Skype, Zoom, браузер и другие приложения, использующие камеру
