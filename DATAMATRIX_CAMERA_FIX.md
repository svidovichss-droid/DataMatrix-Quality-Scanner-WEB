# Исправление проблемы: камера не видит и не захватывает Data Matrix для анализа качества

## Проблема
Камера не распознаёт и не захватывает Data Matrix коды для последующего анализа качества по ГОСТ Р 57302-2016.

## Внесенные изменения

### 1. Улучшена визуализация обнаруженных кодов (`src/ui/main_window.py`)

**Добавлен метод `_draw_detection_overlay()`** (строка 634):
- Отрисовка цветных рамок вокруг найденных Data Matrix кодов
- Цветовая индикация качества:
  - **A (зеленый)**: отличное качество (4.0)
  - **B (светло-зеленый)**: хорошее качество (3.0)
  - **C (желтый)**: удовлетворительное качество (2.0)
  - **D (оранжевый)**: низкое качество (1.0)
  - **F (красный)**: брак (0.0)
- Отображение декодированных данных и статуса PASS/FAIL
- Точка в центре кода для точной локализации

**Обновлен метод `_on_frame_ready()`** (строка 585):
- Сохранение результатов детектирования для визуализации
- Передача данных о найденных кодах в метод отрисовки
- Отображение информации о каждом коде в реальном времени

### 2. Агрессивная предобработка изображений (`scanner/datamatrix_decoder.py`)

**Метод `_preprocess_aggressive()`** (строка 628):
- CLAHE (Contrast Limited Adaptive Histogram Equalization) для улучшения локального контраста
- Бинаризация Оцу для оптимального порога
- Расширенные морфологические операции

**Улучшенные методы декодирования**:
| Метод | Количество вариантов обработки |
|-------|-------------------------------|
| `_detect_with_pylibdmtx()` | 5 (оригинал + 4预处理) |
| `_decode_region_with_pylibdmtx()` | 5 вариантов |
| `_decode_with_pylibdmtx()` | 5 вариантов |
| `_detect_with_pyzbar()` | Автоматическое определение типа |
| `_decode_region_with_pyzbar()` | 5 вариантов |
| `_decode_with_pyzbar()` | 5 вариантов |

### 3. Многоуровневое сканирование (`scanner/datamatrix_decoder.py`)

**Последовательность поиска**:
1. **pylibdmtx** с параметрами shrink 1-5
2. **pyzbar** как альтернативный декодер
3. **OpenCV WeChat QR detector** для сложных случаев
4. **Контурный анализ** для крупных четких кодов

**Варианты предобработки для каждого метода**:
1. Оригинальное изображение
2. Стандартная предобработка (Gaussian + sharpening + adaptive threshold)
3. **Агрессивная предобработка (CLAHE + Otsu)**
4. Инверсия стандартной
5. **Инверсия агрессивной**

## Рекомендации по улучшению захвата

### Освещение
- Используйте равномерное рассеянное освещение
- Избегайте бликов и теней на коде
- Оптимальный угол освещения: 45° к поверхности

### Положение камеры
- Перпендикулярно к поверхности кода (±15°)
- Расстояние: код должен занимать 10-30% кадра
- Фокусное расстояние: четкие границы модулей

### Настройки камеры
- **Экспозиция**: фиксированная, без автоподстройки
- **Gain**: минимальный для снижения шума
- **FPS**: 30+ для движущихся объектов

## Запуск приложения

```bash
python src/main.py
```

### Последовательность действий:
1. Выберите камеру из списка (автоматическое сканирование при запуске)
2. Нажмите **"Подключить"**
3. Нажмите **"ЗАПУСТИТЬ ИНСПЕКЦИЮ"**
4. Наведите камеру на Data Matrix код
5. Наблюдайте результат в реальном времени:
   - Цветная рамка вокруг кода
   - Оценка качества (A-F)
   - Декодированные данные
   - Статус PASS/FAIL

## Примечания

- В тестовой среде (/workspace) физическая камера отсутствует
- На реальной системе код автоматически определит правильный бэкенд
- Поддерживаются Windows (CAP_DSHOW/MSMF), Linux (V4L2), macOS (AVFoundation)
